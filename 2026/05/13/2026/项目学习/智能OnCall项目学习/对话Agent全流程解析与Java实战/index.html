2. ReAct设计模式核心原理

🧠 什么是ReAct？

ReAct = Reasoning + Acting

ReAct是一种让大模型能够边思考边行动的设计模式，通过交替执行推理（Reasoning）和行动（Acting）来完成复杂任务。

🔄 ReAct工作流程

📐 ReAct vs 其他模式

模式 特点 优点 缺点
直接生成 一次性输出答案 速度快 容易幻觉，无法获取实时信息
Chain-of-Thought 思考链推理 逻辑清晰 无法与外部交互
ReAct 思考+行动交替 能调用工具，减少幻觉 需要多次调用，成本较高

7. 面试话术与总结

🎯 面试话术

面试官：”介绍一下你的对话Agent是怎么设计的？”

你：

“我的对话Agent基于ReAct设计模式，核心是思考-行动-观察的循环：

技术实现：

  1. 使用Spring AI Alibaba的ReactAgent框架
  2. 通过**@Tool注解定义工具，支持Function Calling**
  3. 系统提示词约束Agent的行为和可用工具

核心流程：
用户提问 → Agent推理 → 决定是否调用工具 → 调用工具获取数据 → 整合结果 → 返回回答

亮点设计：

  1. 会话管理：ConcurrentHashMap存储会话，支持多轮对话
  2. 历史窗口：限制历史消息数量，避免token溢出
  3. 流式输出：SSE实现逐token返回，提升用户体验
  4. 多Agent协作：AIOps场景使用Supervisor-Planner-Executor架构”
