OhMyOpenCode 技能生态与开发方法论指南

日期: 五月 13, 2026  14:30:00
分类: AI开发

Hi，我是 meisijiya ，也是 老江湖 ，感谢你的阅读，我们一起进步。

在使用 OhMyOpenCode 进行 AI 辅助开发时，理解其技能生态系统和开发方法论的运用，能让你的开发效率提升数倍。本文将深入解析 OhMyOpenCode 的技能系统、SDD/TDD/BDD/DDD 方法论的实践，以及 Superpowers 等框架的支持情况。

1. OhMyOpenCode 技能生态系统

1.1 技能来源与优先级

OhMyOpenCode 的技能系统采用分层架构，不同来源的技能有不同的优先级：

优先级 来源 说明 示例
1️⃣ 最高 User Skills（用户安装） 用户通过 opencode 安装的 skill test-driven-development, brainstorming
2️⃣ 中等 Opencode Skills opencode 平台提供的 skill 平台级技能
3️⃣ 最低 Builtin Skills（内置） OhMyOpenCode 自带的 skill playwright, git-master

关键机制：用户安装的 skill 会覆盖内置默认！

2. SDD/TDD/BDD/DDD 方法论对比

2.1 方法论概览

方法论 全称 核心思想 AI 辅助开发中的角色
SDD Specification-Driven Development 规格先行 定义”做什么”
TDD Test-Driven Development 测试先行 定义”怎么验证”
BDD Behavior-Driven Development 行为先行 定义”用户怎么用”
DDD Domain-Driven Design 领域建模 定义”业务是什么”

2.2 在 Vibecoding 中的融合运用

Vibecoding 的本质是：用自然语言描述意图，AI 理解并实现。

3. Superpowers、Spec-Kit、OpenSpec 支持分析

3.1 Superpowers ✅ 已支持

因为你在opencode安装了这个skill。

Superpowers 已经深度集成到 OhMyOpenCode 的技能系统中！

从 using-superpowers skill 可以看到：

Superpowers 的核心理念：

  • 技能应该在 1% 可能性匹配时就被加载
  • 用户指令永远优先
  • 技能覆盖默认系统行为

在 OhMyOpenCode 中的体现：

  • 技能系统自动匹配和加载
  • 用户安装的 skill 优先级最高
  • 技能内容注入到代理上下文

3.2 Spec-Kit ⚠️ 部分支持（通过等效系统）

Spec-Kit 的核心功能：

  • 规格定义和管理
  • 规格到代码的映射
  • 规格变更追踪

OhMyOpenCode 的等效实现：

Spec-Kit 功能 OhMyOpenCode 等效 说明
规格定义 writing-plans skill 编写结构化计划
规格管理 .sisyphus/plans/*.md 计划文件存储
规格执行 executing-plans skill 按计划执行
规格验证 verification-before-completion 完成前验证

虽然没有直接集成 Spec-Kit，但功能已覆盖！

3.3 OpenSpec ⚠️ 部分支持（通过代理系统）

OpenSpec 的核心功能：

  • 规格澄清和细化
  • 需求分析和验证
  • 规格一致性检查

OhMyOpenCode 的等效实现：

OpenSpec 功能 OhMyOpenCode 等效 说明
规格澄清 Metis 代理 需求分析和歧义识别
需求细化 Brainstorming skill 探索用户意图
规格验证 Momus 代理 计划审查和验证

Metis 代理本质上就是一个 OpenSpec 的实现！

3.4 对比总结

框架 支持程度 OhMyOpenCode 等效 是否需要额外配置
Superpowers ✅ 完全支持 技能系统 无需配置
Spec-Kit ⚠️ 功能覆盖 writing-plans + executing-plans 无需配置
OpenSpec ⚠️ 功能覆盖 Metis + Brainstorming 无需配置

6. 总结

6.1 核心要点

  1. 技能系统是分层的：用户安装的 skill 优先级最高
  2. 方法论已融合：SDD/TDD/BDD/DDD 都在代理系统中体现

6.2 行动建议

6.3 最终建议

OhMyOpenCode 已经内置了 SDD/TDD/BDD/DDD 的精华，你不需要额外配置 Spec-Kit 或 OpenSpec。

它的代理系统（Metis → Prometheus → 执行 → 验证）本质上就是一个混合了所有方法论的智能开发系统。

真正需要做的是：

  1. 理解每个代理的职责
  2. 正确使用 skill 系统
  3. 遵循工作流程而不是跳过步骤
  4. 用正确的提示词触发正确的代理

掌握了这些，你的 AI 辅助开发效率将提升数倍。基础薄弱没关系，在实践中不断优化，很快就能形成肌肉记忆。祝你开发顺利！ 🚀
