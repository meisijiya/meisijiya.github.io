模块1：Linux和shell

一、Linux（Ubuntu）基础

1. 核心概念

  • 与Windows核心差异：Linux是多用户、多任务的开源操作系统，常用于服务器环境。
  • 核心原则：Linux系统中一切皆文件。
  • 默认解析器：Ubuntu默认Shell解析器为bash，可通过echo $SHELL查看，输出为/bin/bash。

2. 系统目录结构

目录 说明
/bin 存放常用命令可执行文件，是/usr/bin的软链接
/sbin 存放系统管理员使用的管理程序，是/usr/sbin的软链接
/home 普通用户主目录，以用户账号命名
/root 超级管理员（root）主目录
/lib 系统开机所需的动态链接共享库，类似Windows的DLL文件
/etc 系统管理所需的配置文件和子目录
/usr 用户程序存放目录，类似Windows的Program Files，包含/usr/bin、/usr/lib等子目录
/boot 存放Linux启动所需的核心文件、连接文件和映射文件
/run 系统运行产生和需要的重要文件
/proc 系统运行进程的虚拟映射目录，可直接访问获取系统信息
/srv 系统运行时服务产生的数据存放目录，默认为空
/sys Linux2.6内核新增文件系统存放目录
/tmp 临时文件存放目录，过期自动清除
/dev 硬件设备的文件形式存储目录，类似Windows设备管理器
/media 系统自动识别的可移动设备（音响、U盘等）挂载目录
/mnt 用户临时挂载外部存储的目录，默认为空
/opt 第三方/可选应用程序安装目录（如MySQL可安装至此），默认为空
/var 存放频繁修改的文件（如系统日志）

（1）基础环境安装

需完成：VMware虚拟机安装、Ubuntu系统安装、Xshell/Xftp远程连接工具安装（生产环境服务器不开放直接接触，需通过远程工具登录）。

（2）APT包管理器（Debian系专用）

APT（Advanced Packaging Tools）可自动完成软件下载、配置、安装，常用命令：

命令 功能

sudo apt install 包名 [-y] 安装软件包，-y跳过确认；如sudo apt install net-tools安装ifconfig工具

  • 切换用户：

二、Shell脚本基础

1. 核心概念

  • Shell是Linux命令解析器，负责解析用户输入的命令，默认解析器为bash，系统支持的所有解析器可通过cat /etc/shells查看，包括sh、bash、rbash、dash等。
  • Shell脚本：多个Linux命令的集合，用于自动化执行重复任务，后缀无强制要求，一般命名为.sh。

方式 语法 说明 权限要求

变量 含义

（2）自定义变量
  • 定义：变量名=值（等号前后无空格；变量名只能由字母、数字、下划线组成，不能以数字开头；值有空格需用单/双引号包裹）
  • 撤销：unset 变量名
  • 只读变量：readonly 变量名=值，只读变量不能修改、不能unset
  • 全局变量：export 变量名，将变量提升为全局环境变量，可供其他Shell程序使用（需在同一进程）
  • 注意：bash中变量默认类型为字符串，无法直接进行数值运算，如C=1+2，echo $C输出1+2。
（3）特殊变量
变量 含义
$n n为数字：$0代表脚本名称，$1-$9代表第1-9个参数，10以上参数需用大括号包裹，如${10}
$# 所有输入参数的个数，常用于判断参数合法性
$* 代表所有参数，被双引号包裹时，所有参数看成一个整体，输出为"$1 $2 ... $n"
$@ 代表所有参数，被双引号包裹时，每个参数单独对待，输出为"$1" "$2" ... "$n"
$? 上一次执行的命令的返回状态：0代表执行成功，非0代表执行失败

10. 正则表达式入门

正则表达式用于描述、匹配一系列符合规则的字符串，grep、sed、awk等命令均支持。

特殊字符 含义 例
^ 匹配行开头 grep ^a /etc/passwd匹配所有a开头的行
$ 匹配行结尾 grep n$ /etc/passwd匹配所有n结尾的行；^$匹配空行
. 匹配任意单个字符 grep r..t /etc/passwd匹配r和t之间有两个任意字符的行
* 匹配前一个字符0次或多次 grep ro*t /etc/passwd匹配rt、rot、root等；.*匹配任意长度任意字符
[] 匹配范围内的单个字符 [68]匹配6或8，[0-9]匹配数字，[a-z]匹配小写字母

二、核心知识点详解

2.1 协程（Coroutine）基础

2.1.1 传统线程/进程的瓶颈

处理I/O密集型任务（如Web服务同时处理数千用户连接）时，线程模型存在明显缺陷：

  • 切换开销大：操作系统在数千线程间切换消耗大量CPU时间
  • 同步复杂：线程共享内存，需用锁保护共享数据，易引发死锁
  • 阻塞浪费：线程I/O阻塞时自身挂起，切换成本仍较高，无法创建足够多线程应对极高并发
2.1.2 协程的定义与优势

协程是Python中处理高并发的强大工具，尤其适合I/O密集型场景：

  • 本质：用户态的轻量级线程（微线程），不由操作系统调度，由程序自身控制调度
  • 核心优势：
    1. 调度可控：通过await关键字主动让出CPU给其他协程
    2. 轻量级：创建成本极低，一个线程可轻松创建数万个协程
    3. 无锁机制：同一时间只有一个协程运行，无需锁保护共享资源
    4. 开销极低：避免操作系统级上下文切换，仅存在极小的函数调用开销，I/O操作效率远高于线程

2.2.2 挂起与恢复：await关键字

await是协程的核心，用于暂停当前协程执行，等待可等待对象（Awaitable，通常是其他协程或I/O操作）完成：

  • 遇到await时：当前协程让出CPU控制权，进入“挂起”状态
  • 事件循环接管：调度执行其他处于“就绪”状态的协程
  • 等待对象完成后：事件循环将控制权交还给挂起的协程，使其从await后继续执行

2.3 事件循环（Event Loop）

事件循环是协程并发模型的核心，是负责调度、管理和执行所有协程的“心脏”。

2.3.1 事件循环工作流程

  1. 注册任务：协程需注册到事件循环才能被调度，三种注册方式：

    注册方式 目的 机制 用途
    asyncio.run(main_coro) 启动整个异步程序 创建EventLoop，将顶级协程隐式包装成Task运行 程序入口点
    asyncio.create_task(coro) 实现并发执行 将协程立即包装成Task，添加到EventLoop队列，后台运行 启动多个并发任务，配合asyncio.gather等待结果
    await coro 实现串行等待 EventLoop隐式将coro注册为任务执行，但必须等待其完成才恢复当前协程 确保代码按顺序执行

  2. 启动协程：通过asyncio.run(main_coroutine)启动，实例化EventLoop，将main_coroutine包装成Task加入循环，进入无限循环选择就绪任务执行。

  3. 遇到await：协程暂停，将I/O操作注册到事件循环监控中，让出控制权给事件循环。

  4. 切换任务：事件循环发现当前协程暂停，立即选择下一个就绪协程执行。

  5. I/O完成通知：I/O操作完成后，操作系统通知事件循环，对应协程标记为“就绪”。

  6. 恢复协程：事件循环在合适时机让挂起的协程从await处恢复执行。

  7. 终止与清理：顶级Task完成后，EventLoop强制取消所有运行中的子Task，关闭循环，程序退出。

2.6 异步函数（async def）与同步函数（def）的选择

FastAPI同时支持异步和同步函数，两者的执行机制和适用场景有明确区别：

2.6.1 异步函数（async def）
  • 本质：定义的是协程（Coroutine），支持在函数内部使用await调用其他异步操作
  • 适用场景：函数内部包含I/O密集型操作（网络请求、文件读写、数据库操作等），且对应操作有异步实现（如aiohttp、异步数据库驱动）
  • 执行机制：FastAPI会直接在事件循环中运行，遇到await时自动让出CPU控制权，事件循环可以切换处理其他请求，实现真正的异步高并发
  • 示例：
2.6.2 同步函数（def）
  • 本质：普通的同步函数，不能在内部使用await，所有操作阻塞当前线程直到完成
  • 适用场景：函数内部是纯计算逻辑（无I/O操作），或使用的库只有同步实现（如旧版数据库驱动）
  • 执行机制：FastAPI会自动将同步函数放入线程池执行（默认线程池大小为os.cpu_count() * 5），多个同步请求会被分配到不同线程并行处理，不会完全阻塞所有请求；但如果同步操作耗时过长（如秒级），会因线程池耗尽导致后续请求排队
  • 示例：
2.6.3 选择原则总结
  1. 只要涉及I/O操作（如数据库、网络请求），优先用async def并配合异步库，发挥FastAPI的异步优势
  2. 纯计算逻辑或无异步库可用时，用def即可，FastAPI会自动处理线程池，无需手动管理
