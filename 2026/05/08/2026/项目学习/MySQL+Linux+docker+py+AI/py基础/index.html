第一阶段

模块1：py的语言类型及执行机制

  1. 两类语言核心区别
  • 编译型：整体翻译成二进制生成可执行文件，直接运行、无源码可见（例：C/C++）。
  • 解释型：逐句翻译即时执行，不生成可执行文件（例：Python、PHP、JS）。

  1. Python执行流程
    先编译生成.pyc字节码（默认仅存内存、不落地），再逐行解释执行字节码。

  2. 与C/Java对比

  • 文件：C的exe、Java的class可见，Python pyc默认不可见。
  • 编译：C/Java需手动编译，Python自动完成编译+解释。
  1. Python类型定位
    表层是解释型语言，底层为先编译后解释的混合执行模式。

模块2：基础语法

核心知识点

1. 代码规范与基础概念

  • 注释：
    • 单行注释：使用 #
    • 多行注释：使用三个单引号 '''注释''' 或三个双引号 """注释"""
  • 缩进：Python严格要求使用缩进（通常4个空格）来表示代码块，替代了Java中的 {}。
  • 标识符命名规则：
    • 只能包含字母、数字和下划线 _，且不能以数字开头。
    • 严格区分大小写。
    • 不可使用Python关键字（可通过 import keyword; print(keyword.kwlist) 查看）。
    • 命名习惯：变量/函数名采用蛇形命名法（如 get_name），类名采用大驼峰命名法（如 MyClass）。

2. 变量与基础数据类型

  • 动态类型：变量无需声明类型，直接赋值即可（如 a = 1），运行时自动推断类型。与Java强类型不同，Python变量可随时改变类型（如 a = 1; a = "hello" 合法）。
  • 基础数据类型：
    • 整型：int（Python3中无Java的long/short之分，长度不受限制）
    • 浮点型：float
    • 布尔型：bool，值为 True 或 False（注意与Java区分：首字母必须大写）
    • 类型检查：使用 type(var) 可查看变量类型。

3. 输入与输出

  • **输出 print()**：默认换行，可通过 end="" 参数设置结尾字符。
  • 输入 input()：接收用户输入，无论输入什么，返回值均为字符串类型，若需数值运算必须强制类型转换（如 int(input("请输入年龄: "))）。

4. 运算符

  • 算术运算符：+ - * /（除法，结果浮点数） %（取模） **（幂运算） //（整除，向下取整，相当于Java的 / 整数除法）
  • 比较运算符：== != > < >= <=，返回布尔值。Python支持连写比较（如 1 < a < 10，Java不支持）。
  • 逻辑运算符（与Java核心差异）：
    • 与：and （对应Java的 &&）
    • 或：or （对应Java的 ||）
    • 非：not （对应Java的 !）
  • 赋值运算符（完整表）：
运算符 说明 实例
= 赋值 a = 1
+= 加法赋值 a += 2，等同于 a = a + 2
-= 减法赋值 a -= 2，等同于 a = a - 2
*= 乘法赋值 a *= 2，等同于 a = a * 2
/= 除法赋值 a /= 2，等同于 a = a / 2
//= 整除赋值 a //= 2，等同于 a = a // 2

:= 海象运算符（Python3.8新增） 表达式中同时赋值并返回值 如 num1 = 20; print((num2 := 3**2) > num1); print(num2)

5. 字符串操作

  • 切片：s[1:3]（取索引1到2的字符，左闭右开），支持负数索引（s[-1] 取最后一个字符）。
  • 常用方法：strip()（去首尾空格）、split()（分割）、join()（拼接）、replace()（替换）、upper()/lower()（大小写转换）。
  • 格式化：推荐使用 f-string 格式 f"Hello {name}"，比Java的字符串拼接或 %s 更直观高效。

6. 流程控制

  • 分支：if..elif..else 结构（**注意：Python使用 elif 替代了Java的 else if**），条件后需加冒号 :。
  • 循环：
    • for i in range(10)：range() 生成整数序列，常用于控制循环次数。
    • for item in iterable：直接遍历可迭代对象（如列表、字符串）。
    • while 循环：满足条件持续执行。
  • 循环控制：break（跳出整个循环）、continue（跳过当次循环），与Java用法一致。

模块3：容器数据类型

核心知识点

1. 列表- 对应Java的ArrayList

  • 特性：可变、允许重复、有序。底层为动态数组。
  • 定义：[] 或 list()
  • 常用操作：
    • 增：append(obj) 末尾追加、insert(index, obj) 指定位置插入、extend(iterable) 合并列表
    • 删：pop(index) 弹出并返回元素（默认最后一个）、remove(obj) 按值删除首个匹配、del list[index] 按索引删除
    • 改：直接赋值 list[index] = new_value
    • 查：list[index] 访问元素、in 关键字判断是否存在（如 if 1 in my_list）
    • 排序：sort() 原地排序、reverse() 原地反转
  • 切片：与字符串切片一致，list[start:stop:step]，如 list[::-1] 快速反转列表。

2. 元组- 对应Java的不可变List

  • 特性：不可变、允许重复、有序。一旦创建，元素和长度均不可修改（安全性强，可作为字典的键）。
  • 定义：() 或 tuple()。注意：单元素元组需加逗号，如 (1,)，否则会被识别为整型。
  • 常用操作：支持索引访问和切片，不支持增删改操作。可使用 count() 和 index() 方法。

3. 字典- 对应Java的HashMap / LinkedHashMap

  • 特性：可变、键不允许重复且不可变（如字符串/数字/元组）、值允许重复、Python 3.7+ 保持插入顺序。
  • 定义：{key: value} 或 dict(key=value)
  • 常用操作：
    • 增/改：dict[key] = value（键存在则修改，不存在则新增）
    • 删：pop(key) 弹出键值对、del dict[key]
    • 查：dict[key]（键不存在抛出KeyError）、dict.get(key, default)（推荐，键不存在返回默认值None或指定值）
    • 遍历：
      • for key in dict: 遍历键
      • for value in dict.values(): 遍历值
      • for key, value in dict.items(): 最常用，同时遍历键值对

4. 集合- 对应Java的HashSet

  • 特性：可变、不允许重复、无序。底层为哈希表，自动去重，常用于成员关系测试和数学运算。
  • 定义：{1, 2, 3} 或 set()。注意：空集合只能用 set()，{} 会被识别为空字典。
  • 常用操作：
    • 增删：add(obj) 添加、remove(obj) 删除（不存在报错）、discard(obj) 删除（不存在不报错）
    • 数学运算：
      • 交集：set1 & set2 或 set1.intersection(set2)
      • 并集：set1 | set2 或 set1.union(set2)
      • 差集：set1 - set2（在set1不在set2）

5. 四类数据结构完整对比

数据结构 是否可变 是否重复 是否有序 定义符号 Java近似对应
列表 可变 允许 有序 [] 或 list() ArrayList
元组 不可变 允许 有序 () 或 tuple() Collections.unmodifiableList
字典 可变 键不允许，值允许 Python 3.7+保持插入顺序 {} 或 dict() LinkedHashMap
集合 可变 不允许 无序 {} 或 set() HashSet

6. 高级特性：推导式
Python特有语法，用于从已知容器快速生成新容器，替代繁琐的 for 循环和 add/append 操作。

  • 列表推导式：

    • 示例：[x**2 for x in range(1, 6)] 生成 [1, 4, 9, 16, 25]
    • 带条件：[x for x in range(1, 11) if x % 2 == 0] 生成1-10的偶数

  • 字典推导式：

    • 示例：{x: x**2 for x in range(1, 4)} 生成 {1: 1, 2: 4, 3: 9}

  • 集合推导式：{表达式 for 变量 in 可迭代对象}

模块5：面向对象

核心知识点

1. 类与对象基础

  • 类定义：使用class关键字定义，语法为class 类名:，无需像Java一样声明访问权限（如public class）。
  • 构造方法：__init__()为Python的构造方法，在创建对象时自动调用，用于初始化实例属性。**第一个参数必须为self**，指代当前实例对象，替代Java中的this关键字（Java的this可省略，Python的self必须显式书写）。
  • 实例方法：类中定义的普通方法，第一个参数默认为self，调用时无需显式传递，Python会自动将调用对象作为self传入，对比Java的实例方法需要依赖对象调用。

2.封装与访问控制：
Java通过public/private/protected三种访问修饰符实现强制封装，Python无原生强制访问控制，仅通过约定和名称改写实现伪封装：

  • **单下划线_var**：约定为非公开API，提示外部不要直接访问，但无强制约束力，外部仍可直接访问。

  • **双下划线__var**：会触发名称改写机制，变量名会被改写为_类名__var，仅类内部可直接访问，外部需通过改写后的名称访问，有一定封装性。

模块6：高级特性

核心知识点

1. 装饰器

  • Python内容：使用@语法定义，本质是一个函数，用于增强其他函数的功能，无需修改原函数代码即可添加额外逻辑。例如@staticmethod用于定义静态方法，逻辑上属于类但无需访问类或实例属性。
  • 与Java对比：Java没有原生的装饰器语法，若要实现类似“在不修改方法代码的前提下添加额外逻辑（如日志、计时）”的功能，需使用AOP（面向切面编程，如Spring AOP）或动态代理（InvocationHandler）实现，配置流程复杂；Python装饰器是语法级原生支持，通过@装饰器名即可直接添加，实现更简洁。

2. 上下文管理器

  • Python内容：通过with语句使用，可自动管理资源（如文件、数据库连接等），无需手动释放资源，避免资源泄露。例如使用with语句打开文件时，会在代码块执行完毕后自动关闭文件，无需手动调用关闭方法。
  • 与Java对比：Java 7+提供了try-with-resources语法，也可自动关闭实现了AutoCloseable接口的资源，和Python的with功能类似；Python的with语句使用更简洁，无需强制要求资源类实现特定接口。

3. 异常处理

  • Python内容：使用try..except..finally结构捕获和处理异常：try块中放可能抛出异常的代码，except块捕获指定类型的异常并处理，finally块中的代码无论是否发生异常都会执行（通常用于释放资源）。
  • 与Java对比：Java异常分为受检异常（必须捕获或在方法上声明抛出）和非受检异常，Python所有异常均为非受检异常，无需强制捕获或声明；Java使用try..catch..finally结构，Python的except对应Java的catch，且Python支持在一个except中捕获多个异常类型。

4. 浅拷贝与深拷贝

  • Python内容：

    • 直接赋值：仅创建对象的引用（别名），不会产生新的拷贝，原对象和赋值对象指向同一块内存空间。

    • 浅拷贝：仅拷贝父对象，不会拷贝内部的子对象，拷贝后仅第一层数据独立，子对象仍共享。实现方式：切片操作[:]、工厂函数list()/set()、copy.copy()。

    • 深拷贝：完全拷贝父对象及所有层级的子对象，拷贝后所有数据均独立，修改原对象不会影响深拷贝对象。实现方式：copy.deepcopy()。

  • 与Java对比：Java中对象赋值均为引用传递，没有原生的深浅拷贝区分；若要浅拷贝对象需实现Cloneable接口并重写clone()方法，深拷贝需手动递归拷贝所有层级的属性，实现成本高；Python原生提供copy模块支持深浅拷贝，使用更便捷。

模块7：网络编程基础

核心知识点

1. 网络编程三要素
网络编程的核心基础概念，所有网络通信均需基于以下三要素实现定位与交互：

  • IP：网络中计算机的唯一标识，用于定位网络中的目标主机。
  • 端口：进程的逻辑地址，用于标识计算机内的指定进程，实现同一主机内不同网络进程的区别。
  • 协议：网络通信的规则集合，明确通信双方的数据交互规范。

2. TCP/IP协议族
现代互联网的核心通信协议族，本模块需掌握的核心内容：

  • 核心协议：包含两大基础协议：
    • TCP（传输控制协议）
    • IP（网际协议）
  • 分层模型：
    • 协议原生分层：应用层、传输层、网络层、网络接口层。
    • 学习常用分层：实际学习时通常替换为OSI七层模型中的数据链路层+物理层，合并为五层模型（应用层、传输层、网络层、数据链路层、物理层）。

3. 通信协议的定义与作用

1. SQL语句分类

SQL（结构化查询语言）是用于操作关系型数据库的标准语言，按功能分为以下四类：

SQL分类 全称 作用 核心指令
DDL 数据定义语句（Data Definition Language） 定义/修改/删除数据库、表、字段等对象结构 CREATE（创建）、ALTER（修改）、DROP（删除）
DML 数据操作语句（Data Manipulation Language） 新增/删除/修改/查询表中数据 INSERT（增）、DELETE（删）、UPDATE（改）、SELECT（查）
DQL 数据查询语句（Data Query Language） 从DML中单独拆分的查询子类（因SELECT使用频率极高） SELECT（查）
DCL 数据控制语句（Data Control Language） 管理数据库访问权限、事务提交回滚 GRANT（授权）、COMMIT（提交事务）、ROLLBACK（回滚事务）

模块3：ORM框架SQLAlchemy

官方文档：https://sqlalchemy.org.cn/

一、整体架构

  • 核心架构

    采用分层架构，分三大核心层级，层级职责清晰：

    • 连接管理层：核心是 Engine（引擎），负责数据库连接；提供 Connection、Result 对象；ORM 层封装 Session 作为操作入口。
    • 元数据结构层：核心为数据库元数据，包含 MetaData（库）、Table（表）、Column（列）三大基础对象，是 SQL 查询的基础。
    • 查询构建层：Core/ORM 共享 SQL 表达式语言，支持流畅构建 SQL，兼容 text () 编写原生 SQL。

  • 两大使用风格

    Core 风格：直接用 Engine+Connection，结合元数据 / SQL 表达式操作，贴近原生 SQL。
    ORM 风格：通过 Session 操作，基于面向对象方式处理数据，适配 OOP 编程场景。

  • 关键特性

    两种风格共享同一套元数据，均支持与 SQLite 等数据库交互。

    总结：

    • 分层架构：连接管理→元数据→查询构建，层层支撑；
    • 双模式通用：Core 贴近 SQL、ORM 适配面向对象，可共用元数据。

二、核心组件

  1. 引擎（Engine）：整个库的基础入口，负责管理数据库连接。
  2. 元数据三基石：
    • MetaData：数据库元数据的容器，管理所有表对象；
    • Table：代表数据库中的表对象；
    • Column：代表数据库表中的列对象。
      三者是Core和ORM的核心要素，统称为数据库元数据，是SQL表达式语言的基础。
  3. SQL表达式语言：SQLAlchemy Core和ORM的核心要素，支持流畅、可组合构建SQL查询，是日常开发的首选查询方式。
  4. text()构造：用于编写文本SQL的简单工具，日常使用中是例外场景，但始终可用，适合未熟悉表达式语言时作为补充。
  5. Core层操作对象：
    • Connection：Engine的主要操作端点，用于Core风格的数据库连接与操作；
    • Result：SQL执行后的结果对象，用于获取返回数据。
  6. ORM层操作对象：
    • Session：ORM对底层Connection等对象的外观模式封装，是ORM风格的核心操作端点，负责管理事务和对象状态；
    • ORM面向表的构造：与Core共享元数据逻辑，提供ORM风格的表定义方式；
    • ORM关联对象：用于处理ORM对象之间的关联关系。

三、环境准备

根据教程顺序，使用SQLAlchemy 2.0前需完成以下准备：

  1. 创建引擎：完成连接配置，初始化Engine对象，是所有操作的基础。
  2. 准备数据库：需确保目标数据库就绪，其他数据库需对应DBAPI支持。
  3. 构建表构造：准备好Core风格的元数据对象（MetaData、Table、Column）或ORM风格的面向表构造，作为数据库交互的基础。
  4. 可选准备：
    • 暂未掌握SQL表达式语言时，可先了解text()构造的使用方法，编写文本SQL完成简单操作；
    • 建议熟悉基础后优先学习SQL表达式语言，文本SQL仅作为补充手段。

四、基本使用流程

  1. 建立连接：创建Engine对象。
  2. 熟悉基础操作：了解Engine的端点Connection、Result，以及ORM的Session对象，掌握事务基本逻辑，可先通过text()构造编写文本SQL完成简单操作。
  3. 使用元数据：构建/使用已有的MetaData、Table、Column等元数据对象，明确Core和ORM两种风格的表构造使用方式。
  4. 数据处理：基于元数据和SQL表达式（或text()文本SQL），通过Connection（Core风格）或Session（ORM风格）执行数据操作，包括INSERT、查询、更新、删除等。
  5. ORM进阶：若使用ORM风格，可进一步学习ORM关联对象的使用，处理对象间关联关系。
    核心交互逻辑：
  • Core风格：Engine → 获取Connection → 执行SQL（表达式/text()） → 获取Result → 处理结果
  • ORM风格：Engine → 获取Session → 操作ORM对象/执行查询 → 提交事务 → 处理结果

五、高级特性

  1. 可组合的SQL表达式语言：支持流畅、可组合的方式构建复杂SQL查询，是SQLAlchemy的核心特性，优于文本SQL，建议在熟悉基础后优先使用。
  2. ORM关联对象处理：支持定义和操作ORM对象之间的关联关系。
  3. 双风格兼容：Core和ORM两种风格共享同一套元数据对象（MetaData、Table、Column），可根据场景灵活切换，无需重复定义表结构。
  4. 文本SQL补充支持：text()构造始终可用，可在特殊场景下编写文本SQL，作为表达式语言的补充，兼容性更强。
