Java常用数据结构及API

作者: 老江湖
日期: 五月 12, 2026  11:17:12
分类: 算法

Hi，我是 meisijiya ，也是 老江湖 ，感谢你的阅读，我们一起进步。

在你开始刷 LeetCode Hot100、准备大厂面试时，掌握 Java 常用数据结构及它们的 API 会让刷题效率翻倍。下面我会从“零基础友好”的角度，带你逐个梳理这些数据结构，给出最常用的方法、代码示例和刷题中的典型应用场景。

1. 数组（Array）

数组是最底层、最快的容器，适合存储固定数量、类型相同的数据。

一维数组

二维数组

Arrays 工具类（必背）

时间复杂度： 访问 O(1)，排序 O(n log n)，查找 O(n)。

常见题： 两数之和（暴力或配合哈希）、二分查找、滑动窗口等。

2. 动态数组 ArrayList

ArrayList 是可变长度的数组实现。如果要频繁随机访问，用它；但如果频繁在头部增删，不建议用。

时间复杂度： 访问/修改 O(1)，尾部插入均摊 O(1)，中间插入/删除 O(n)。

3. 字符串（String & StringBuilder）

String (不可变)

StringBuilder (可变，非线程安全)

常用场景： 频繁拼接字符串（代替 + 号），动态构建结果。

5. 栈与队列

6. 哈希表（HashMap & HashSet）

几乎 70% 的题都会用到，利用 O(1) 均摊时间快速查找。

典型题： 两数之和、存在重复元素、无重复字符的最长子串、字母异位词分组。

9. 图（Graph）

图在 LeetCode 中常以两种形式出现：邻接表 或 邻接矩阵。

邻接表（推荐，省空间）

BFS / DFS 图遍历

  • BFS 用 Queue + boolean[] visited
  • DFS 用递归或显式 Deque 栈

邻接矩阵： 一般直接 int[][] matrix，适用于稠密图。

典型题： 岛屿数量、课程表（拓扑排序）、克隆图。

11. 面试速查表（根据场景选结构）

场景 选择数据结构 原因
快速查找是否存在 HashSet / HashMap O(1)
需要键值计数、分组 HashMap O(1) 映射
维护最大/最小值 PriorityQueue（堆） O(log n) 插入删除，O(1) 取最值
后进先出/括号匹配/回溯 Deque（栈） push/pop O(1)
层序/广度优先 Queue（BFS） 先进先出
固定长度序列随机访问 Array / ArrayList O(1) 访问
频繁头部增删 LinkedList（队列用）或 Deque O(1) 头尾操作
滑动窗口最大/最小值 单调队列（Deque） 均摊 O(1)
前缀、双指针遍历字符串 String + toCharArray 方便修改字符
构造结果字符串 StringBuilder 避免大量 String 拼接
动态集合求最值、中位数 堆（或双堆） Top K / 中位数

把这些容器的常用 API 牢记，配合一些题目练习，你会发现代码越写越快、思路也更容易落地。基础薄弱没关系，在刷题时有意识地查阅并反复使用，很快就能形成肌肉记忆。祝你刷题顺利，面试拿下！
