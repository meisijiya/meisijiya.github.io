现在AI生成的前端UI同质化严重，也许用上官方的各种前端UI-skill能解决问题，但是有没有个性化的UI生成方案呢，有的有的。

OpenCode Skills 安装完全攻略

准备：

  • Dribbble上的设计稿或者某个产品页面
  • https://stitch.withgoogle.com/ 图生代码工具
  • opencode 并安装 /skill-creator 技能

详细操作请看博客，我这里简要说明一下，就是通过geogle的这个工具将心仪的UI参考图给到它，然后将生成的代码下载并拷贝到目标目录，接着在opencode使用/skill-creator 将代码设计成一个UI-skill。

搞定后就可以使用这个skill来辅助前端UI界面的开发了。

当然我们也可以使用**ui-ux-pro-max-skill**这个UI/UX设计skill进行前端原型设计，设计好原型就可以让AI根据该原型使用creater-skill技能来创建对应skill进行前端UI开发。

[!tip]

这个skill实际上：是一款 AI 技能工具，依托推理引擎自动生成适配行业的完整设计系统，快速输出专业 UI/UX 方案，覆盖网页、移动端、跨平台等场景。

应当注意：

  • 核心并非提升 AI 写代码能力，而是补齐其设计系统与规范理解能力。
  • 输出质量取决于 AI 助手、模型及 Skill（知识库 + 工作流），推理效果由模型决定。
  • 建议搭配质量审核，避免设计理念在落地实现中偏离。

官方文档推荐CLI全局安装npm install -g uipro-cli，安装后需要为为特定AI助手初始化uipro init --ai opencode，opencode使用这个命令即可。

初始化成功后，CLI 会自动：

  • 在项目根目录创建.uipro配置文件夹
  • 生成适配所选 AI 助手的配置文件
  • 安装必要的依赖包
  • 配置 AI 助手的技能触发规则

接着重启opencode就可以看到/skills中的ui-ux-pro-max技能。

[!note]

使用技巧：

可以/skills选择这个技能，然后大概设计一个布局图给到AI（能够识图的大模型或者带有识图的MCP服务），给到AI你的项目需求（根目录创建一个，需求文档尽可能的覆盖你的核心需求），可以描述一下项目的风格类似于什么（控制台、电商平台等），然后在颜色大全网站选择几个喜欢的颜色粘贴十六进制色值给到AI。也就是说给到AI这几样东西：

  • 大概布局图
  • 需求文档
  • 风格
  • 主题颜色

然后设计好的原型图进行审查，不满意就聊到满意为止；

设计好的原型图就可以交给creater-skill来设计一个skill，之后就可以使用该技能进行前端的实现。

第二章：项目进行时

claude-code源码学习仓库

2026.3.31ClaudeCode源码泄露，作为全球最强 AI 编程工具，源码里的核心设计，每一个都值得学习和借鉴，具体如下：

  • Agent 循环引擎：采用基于 ReAct 机制的 Agent 循环引擎，是核心执行逻辑。

  • 内置工具安全设计：搭载 40 多个内置工具，有完善的安全设计规则。

  • 工具并发机制：采用读写分离的工具并发机制，提升执行效率。

  • 系统提示词优化：对系统提示词做缓存分裂优化，降低资源消耗。

  • 内容检索策略：无需依赖 RAG 即可实现高效内容检索。

  • 三层分级记忆：搭建三层分级记忆架构，优化信息存储与调用。

  • 上下文压缩：使用五级上下文压缩，精简对话信息提升响应速度。

  • Bash 安全检查：设置严格的 Bash 安全检查规则，规避执行风险。

  • 卧底模式：内置卧底模式，属于特殊功能设计。

当然除此之外，ClaudeCode完全实践并实现了 Harness（驾驭层 / 控制体系）的完整理念，是目前 AI Agent 领域 Harness 工程化的标杆。那么Harness Engineering是什么：

  • Harness Engineering核心定义：是模型外的运行系统，决定AI系统能否稳定运行，Agent=Model+Harness，Harness=Agent-Model。
  • AI工程三次重心迁移：从Prompt Engineering（把任务讲清楚）→Context Engineering（把信息给对）→Harness Engineering（让模型持续做对），三者是包含关系，范围逐层扩大。
  • Prompt Engineering核心：通过语言设计塑造模型概率空间，解决模型听懂指令的问题，存在无法弥补知识缺失的天花板。
  • Context Engineering核心：在合适时机给模型正确信息，涵盖输入、历史、检索结果等所有决策信息，RAG、Agent Skills是典型实践。
  • Harness Engineering六层架构：1.上下文管理（规范模型思考边界）；2.工具系统（管控工具调用与结果）；3.执行编排（串联任务步骤）；4.记忆状态（区分任务、中间结果、长期记忆）；5.评估观测（验证输出与错误归因）；6.约束恢复（校验纠错、失败重试）。
  • Anthropic实践案例：用Context Reset解决上下文溢出，拆分Planner/Generator/Evaluator实现生产验收分离，解决自评失真问题。
  • OpenAI实践案例：人类工程师负责设计环境、拆解任务、建立反馈，采用渐进式披露信息，让Agent自主验证、修复问题，形成自动治理系统。
  • Harness Engineering价值：解决模型长链路执行稳定性问题，是AI落地的关键，决定系统能否稳定交付而非仅模型能力。

这给我们一个很好的启发，对于我们开发全栈项目而言就是：

  • AI 不负责做架构决策、定业务逻辑、把控安全
  • AI 只负责写代码、生成脚本、查报错、拼页面、写配置
  • 你要给 AI 划定：技术栈、目录规范、接口格式、权限边界、任务范围

第一章搞完后，会发现思路清晰了，只需要跟AI商讨好功能，然后生成直至满意的代码。

那有没有什么需要注意的呢，有的有的，AI一旦达到上下文窗口的某个阈值就会癫癫的，因此通常需要拆解各个功能模块，采取增量法，慢慢完善实现，将每次生成的代码都生成一个总结日志来记录AI的操作以及我们的提问，也更好的为我们复盘，我们可以在项目的根目录创建AGENTS.md：

此外，若后续需要将该项目经历整理写入简历，还可以在每完成一个完整功能模块后，让 AI 对该功能的整体实现流程、技术要点与完成成果进行系统性总结，为简历撰写提炼出清晰、专业的项目描述内容；也可以生成对应的八股文剖析讲解。

写项目具体过程怎么做呢，以下是功能开发的一点思路：

  1. 优先开发独立性强的功能；

  2. 首先口述需求，将对应的表生成出来；

  3. 然后进行后端MVC全自动生成，生成顺序如下（可以拆成一步一步去跟AI交涉来完善细节）：

    必要基础骨架：统一返回 Result、全局异常、JWT、权限拦截、 权限

    1. Entity 实体类
    2. Mapper 接口
    3. Service 接口 + impl
    4. Controller（REST 接口）
    5. Swagger / OpenAPI + 接口测试
    6. AI 代码审查 + 测试 → 没问题再给前端、出接口文档

  4. 前端先利用自己生成的UI-skill或者官方UI-skill或者搭配组件库来构建前端页面，然后再逐个对接后端的API；

  5. 最后前后端联调测试；

其实有了AI，就很考验我们的技术栈的广泛了解程度以及如何搭配的技巧，业务设计以及各种调优策略，展开来说就是：

  1. 技术栈广度与选型能力
    知道什么场景该用什么技术，什么时候上缓存、消息队列、分库分表，什么时候用异步、事件驱动，而不是只会一套 CRUD。

  2. 架构与业务设计能力
    需求怎么拆解、边界怎么划、怎么高内聚低耦合、怎么兼顾扩展性和可维护性。AI 能帮你实现细节，但设计思路得你自己给。

  3. 排查、调优与深度理解
    性能瓶颈、内存泄漏、死锁、GC、SQL 优化、JVM 参数、网络调优……
    AI 可以给方案，但你得能判断对不对、能不能落地、为什么会这样。

  4. 把 AI 当成工具链的一环
    不是依赖 AI 写代码，而是用 AI 加速自己的思考和验证：

    • 快速验证思路
    • 生成样板代码
    • 查冷门配置和异常
    • 辅助做方案对比

所学即所用，利用AI来快速学习一门技术栈是当前时代所必备的。

后续将自己准备好的语料库跟AI battle就好了

如何了解一门技术栈或者说如何了解一个功能到底要怎么实现呢，很显然得去网上找资料，当然不是一股脑看，而是让AI总结大纲了解大概，细节对于我们来说可以在写项目的过程中慢慢领悟，利用plan模式跟AI确定并了解实现细节可以事半功倍。

我建议网上看到的什么功能讲解的博客，可以用以下提示词来归纳实现流程，以达到复现的目的：

prompt提示词技巧

基础技巧：

  1. 明确指定任务和角色；
  2. 提供相似说明和具体示例；
  3. 使用结构化格式引导思维；
  4. 明确输出格式要求；

进阶技巧：

  1. 思维链提示；
  2. 少样本学习；
  3. 分步骤指导；
  4. 自我评估和修正；
  5. 知识检索和引用；
  6. 多视角分析；
  7. 多维态思维；

这么多技巧怎么使用呢，很简单，把你的原始prompt给到AI，然后粘贴基础技巧给到AI，按需选择进阶技巧，最后让AI来完善prompt即可，然后审查，对生成的调试次进行调试与优化：

  1. 迭代式提示优化；
  2. 边界测试（通过极情况测试模型的能力边界）；
  3. 提示词模板化；
  4. 错误分析与修正；
