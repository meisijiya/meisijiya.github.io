第11章 基于AgentScope+Nacos的分布式多智能体旅游规划系统实战

📋 章节概述

本章基于 **AgentScope 1.0（Java版）+ Spring Boot 4.0.2 + Nacos 3.1.0+**（生产环境推荐稳定版3.2.0），完整落地分布式旅游规划多智能体系统，覆盖从架构设计、智能体注册发现（A2A协议）、任务分发执行、MCP工具集成、自主调度到人工介入的全流程。

结合Nacos 3.1.0里程碑版本的A2A注册中心、MCP Registry核心能力，以及AgentScope的Plan、Hook、A2A、MCP等特性，实现多智能体协作的AI应用落地。

注：AgentScope内置轻量级单机A2A注册中心，仅适用于本地测试；分布式场景需使用Nacos作为中心化注册中心，实现跨节点Agent管理。

11-1 回顾AgentScope旅游规划整体架构

核心背景

随着AI技术发展，A2A（Agent-to-Agent）协议成为多智能体协作的标准，Skills（技能）特性成为智能体能力描述的核心维度，本章项目同步整合Skills特性，强化智能体任务处理的专业性。

项目技术栈与模块划分

模块 作用 技术依赖
公共模块 存放通用工具类（智能体构建、Nacos客户端生成、工具注册、Plan管理） AgentScope Core、Nacos Client
主管Agent模块 接收用户需求、分解任务、调度远程智能体 AgentScope Plan、Hook、A2A Client
路线制定Agent模块 执行自驾游路线规划任务，集成百度地图MCP工具 AgentScope MCP、A2A Server
行程规划Agent模块 执行景点行程规划任务 AgentScope A2A Server

核心协作流程

AgentScope核心能力支撑

  • Plan：主管Agent通过PlanNotebook实现复杂任务自主分解，支持用户确认、历史计划恢复（对应官网《Plan - AgentScope Java》文档）
  • A2A协议：基于Google捐赠给Linux基金会的A2A开放标准，实现Agent间通信协作（对应官网《A2A (Agent2Agent) - AgentScope Java》文档）
  • MCP：集成Model Context Protocol，实现外部工具（如百度地图）的标准化接入（对应官网《MCP (Model Context Protocol) - AgentScope Java》文档）

11-2 测试团队成员基于A2A协议注册到Nacos（Nexus）

Agent注册两种方式

AgentScope支持两种Agent注册到A2A中心的方式，本项目选择自动注入方式适配分布式场景：

注册方式 适用场景 实现逻辑
自动注入（推荐） 分布式项目 将Agent声明为Spring @Bean，引入Nacos注册依赖，Spring Boot自动将AgentCard注册到Nacos
手动写入 本地测试/AgentScope自带注册中心 通过AgentScopeA2aServer构建服务端，手动写入AgentCard和部署信息，需手动启动注册中心

Nacos A2A注册中心规范（官网补充）

  • 命名空间隔离：AgentCard仅属于单个命名空间，同一命名空间内Agent名称不可重复，不同命名空间可重名
  • 名称限制：Agent名称长度≤64字符，仅支持ASCII可视字符（编码32~126）
  • 版本管理：同一Agent可存在多个版本，可设置默认发布版本，使用者可指定版本获取

11-5 获取百度地图MCP服务端的工具列表

MCP核心逻辑

MCP（Model Context Protocol）是工具接入的标准协议，需客户端与服务端配对：

  1. 客户端与服务端建立会话连接
  2. 客户端从服务端获取工具列表
  3. 工具注册到Agent的Toolkit中供调用

百度地图MCP客户端实现

1. 传输协议匹配（关键）

客户端必须与服务端使用相同的传输协议，否则无法建立连接：

传输类型 适用场景 状态
StdIO 本地进程通信 有状态
SSE HTTP流式传输 有状态
StreamableHTTP 无状态HTTP传输 无状态

百度地图MCP官方服务端地址从魔塔社区获取，选择对应传输类型（本项目使用SSE）。

2. 客户端创建与初始化

验证结果

11-8 人工介入修改主管Agent制定的计划

核心需求

主管Agent制定的计划可能不符合用户预期，需支持用户在计划生成后、执行前介入修改，而非仅做「是/否」确认。

实现方案

结合AgentScope的UserAgent（内置用户交互Agent，无需大模型）和Hook（执行阶段扩展点）实现：

  1. 用PlanNotebook生成计划，监听PreReasoningEvent事件
  2. 计划生成后打断主管Agent执行，调用UserAgent接收用户输入的修改意见
  3. 主管Agent根据意见调整计划后继续执行

📝 章节总结与注意事项

版本兼容性

组件 最低版本 推荐版本
Nacos 3.1.0 3.2.0（生产环境）
AgentScope 1.0 1.0.0.4+
Spring AI Alibaba 1.0.0.3 1.0.0.4+
JDK 17 17+

常见问题

  1. Agent注册报错：检查依赖是否仅放在需要注册的模块，避免无需注册的Agent（如主管Agent）引入注册依赖
  2. MCP连接失败：检查客户端与服务端传输协议是否一致，API Key是否正确
  3. 任务调用类型报错：统一使用AgentBase作为Agent类型参数，兼容本地和远程Agent
  4. 工具未加载：MCP客户端注册到Toolkit时需加block()阻塞，等待工具加载完成

扩展方向

  • 利用AgentScope Skills特性，实现更精细的Agent能力匹配
  • 结合Nacos MCP Router，实现MCP服务的智能路由与权限控制
