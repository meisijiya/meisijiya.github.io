第12章 AgentScope实践：行程规划智能体Skills集成与SubAgent搭建

本章整合2025年11月实践文稿与AgentScope 2026年最新官方文档，从0到1实现具备景点推荐、表格制作能力的行程规划智能体，涵盖Skill编写、挂载、渐进式加载，以及SubAgent拆分的全流程。

12.1 行程规划智能体挂载Skills基础（对应原12-1文稿）

12.1.1 背景与目标

此前已完成：

  • 主管Agent可通过Nexus发现远程路线制定Agent、行程规划Agent，根据智能体卡片描述分配任务
  • 路线制定Agent已挂载百度地图MCP，可调用服务端工具执行任务

本节目标：让行程规划Agent挂载多个专业Skill（如景点推荐、表格制作），具备专项执行能力。

12.1.2 Skill核心依赖

⚠️ 文稿中笔误说明：skill.markdown/agents kill 官方规范为 SKILL.md/AgentSkill，以下内容以官方术语为准。

每个Skill是独立模块化的能力包，核心依赖：

  1. 必填文件：SKILL.md（位于Skill文件夹根目录）

  2. 必填字段（文稿+官网强制要求）：

    字段 要求 说明
    name 必须与Skill文件夹名称一致，全小写+数字+下划线 技能唯一标识
    description 清晰说明触发场景与使用条件 供智能体判断是否需要激活该Skill

  3. 工程实践建议：优先采用文件加载或远程仓库加载方式，避免手动创建对象的复杂性（文稿+官网共同建议）

12.1.3 AgentSkill对象创建方式（3种）

若需手动创建AgentSkill对象，支持以下3种方式，均不推荐用于工程实践：

方式1：Builder工厂模式

方式2：Markdown字符串转换

方式3：直接构造方法

12.1.4 推荐：文件/仓库加载方式（工程首选）

文稿与官网均建议避免手动创建对象，优先使用以下标准化加载方式：

加载方式 适用场景 示例代码
Classpath文件加载 JAR包/Spring Boot项目，本地预打包Skill ClasspathSkillRepository repo = new ClasspathSkillRepository("skills"); List<AgentSkill> allSkills = repo.getAllSkills();
Git仓库加载 只读远程Skill，团队共享 GitSkillRepository repo = new GitSkillRepository("https://github.com/xxx/skills.git");
Nacos仓库加载 在线场景，实时感知Skill变更 NacosSkillRepository repo = new NacosSkillRepository(aiService, "namespace");
MySQL持久化 多环境共享，重启后保持可用 MysqlSkillRepository repo = new MysqlSkillRepository(dataSource, true, true);

📢 文稿提到的1.0.8版本JarSkillRepositoryAdapter已废弃，1.0.9+版本统一使用ClasspathSkillRepository。

12.2.2 SKILL.md编写要求

① 必填YAML头（强制要求）

② 建议补充内容（文稿实践经验，避免Skill无核心能力报错）

  1. 核心能力说明：
    • 能力1：免费景点推荐（自然风光、历史古迹，提供最佳游览时间、避峰建议）
    • 能力2：高性价比美食推荐（本地特色小吃、隐秘餐厅，标注价格区间）
    • 能力3：小众打卡点推荐（网红拍照点、出片秘密景点）
    • 能力4：住宿方案建议（经济型/舒适型两种方案，标注价格、位置）
  2. 工作流程：综合距离、时间、费用、用户偏好，输出含景点、美食、住宿的完整方案。
  3. 回答约束：
    • 满足可行性、费用约束
    • 提供具体地址、价格区间、开放时间
    • 结合实时天气预报给出建议

12.3 表格制作Skill集成实践

12.3.1 第三方Skill获取：避免重复造轮子

文稿实践经验：主流Skill（表格制作、PDF操作、浏览器控制等）无需自行编写，可从Skills聚合平台获取现成资源（例如有赞Skills聚合平台），直接下载适配版本即可。

12.3.2 表格制作Skill核心能力

  • 支持Excel（.xlsx）、CSV格式表格的创建、编辑
  • 支持表格数据分析、格式设计
  • 配套脚本：Python/Java脚本实现表格操作逻辑

12.3.3 版本差异说明

框架版本 资源加载类 说明
1.0.8（文稿使用版本） JarSkillRepositoryAdapter（已废弃） 从classpath资源文件夹加载Skills，该类在1.0.9+版本移除
1.0.9+（官网最新版本） ClasspathSkillRepository 替代废弃类，兼容标准JAR、Spring Boot Fat JAR

12.4 Skills与工具渐进式加载实现（对应原12-4文稿）

12.4.1 渐进式加载的核心价值

智能体上下文窗口有限，传统方式一次性加载所有Skill的完整资源（脚本、文档、工具）会占用大量上下文。渐进式加载实现：

  • 未使用时：智能体仅感知Skill的名称、描述等元数据（~100tokens/Skill）
  • 需要时：通过read_skill/load_skill_through_path工具加载完整SKILL.md内容（<5k tokens），同时激活绑定工具
    （来源：官网《Skills（渐进式披露）》文档）

12.4.2 Skill与Tool结合实现步骤

  1. SkillBox关联Toolkit：Toolkit是工具包载体，SkillBox通过Toolkit实现工具管理

  2. 注册Skill到SkillBox：

  3. 绑定Tool到Skill：将Skill依赖的工具（如天气查询工具）注册到SkillBox，实现工具随Skill激活而加载

  4. 智能体注入Toolkit：智能体通过Toolkit感知拥有的工具，仅在Skill激活时加载完整工具逻辑

12.5 景点推荐SubAgent拆分实践

12.5.1 为什么拆SubAgent？

行程规划Agent作为主智能体，若承载所有任务会导致上下文混乱。将单一任务（如景点推荐）拆为独立SubAgent：

  • SubAgent拥有独立的上下文、工具、Skill，执行任务后仅返回结果给主Agent，避免主上下文膨胀
  • 主Agent（行程规划）通过Task工具调用SubAgent，无需感知SubAgent内部实现
    （来源：官网《Subagents（子智能体）》文档）

12.6 完整架构总结

12.7 常见问题与最佳实践

12.7.1 易错点提醒

  1. name字段需与Skill文件夹名称、SKILL.md的name完全一致，否则加载失败
  2. 文稿中skill.markdown需改为官方规范的SKILL.md
  3. 1.0.8版本升级到1.0.9+需将JarSkillRepositoryAdapter替换为ClasspathSkillRepository

12.7.2 性能优化建议

  1. SKILL.md控制在1.5-2k tokens，详细文档放在references/文件夹
  2. 主流Skill优先从聚合平台获取，减少重复开发
  3. 定期清理旧版本Skill：skillBox.clearSkillOldVersions()
  4. 避免重复注册相同Skill，框架自带重复注册保护机制

12.7.3 官方最佳实践

  1. SubAgent仅承载单一任务，避免职责过多

  2. Markdown定义的SubAgent/Skill要纳入版本控制，方便团队协作

  3. 开启Skill渐进式披露，避免上下文窗口溢出

  4. SubAgent若需调用外部服务（如百度地图MCP），可单独挂载MCP工具，与路线制定Agent保持一致
