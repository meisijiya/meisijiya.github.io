Spring AI Alibaba 1.1 核心实战：智能体开发、MCP协议与A2A通信

✅ 1.1相比1.0版本核心定位：聚焦上下文工程、多智能体协同协作、复杂工作流，将智能体开发作为核心方向，新增agent-framework依赖，原生支持工具调用、大模型交互、MCP内置集成与A2A协议，建议学习路线优先跟进1.1版本，后续适配2.0迭代。

一、Spring AI Alibaba 1.1 依赖管理

1.1版本仍支持BOM（Bill of Materials）方式统一管理依赖版本，避免版本冲突，所有BOM必须放在<dependencyManagement>标签中。

1.1 版本说明

  • 2025.10稳定版：1.1.0-RC2，同步支持Spring AI官方1.1版本依赖
  • 需同时引入两套BOM：
    1. 阿里系依赖BOM：com.alibaba.cloud.ai:spring-ai-alibaba-bom，管理所有阿里云AI相关依赖
    2. Spring AI官方BOM：org.springframework.ai:spring-ai-bom，管理mcp-server-webflux等Spring官方AI依赖（非阿里包）

1.2 核心依赖说明

依赖 artifactId 作用 版本说明
spring-ai-alibaba-agent-framework 1.1新增核心智能体框架，支持工具调用、大模型交互、智能体开发 BOM管理，无需指定版本
spring-ai-alibaba-a2a-nexus A2A协议注册中心依赖，对接Nexus 3.1 BOM管理，无需指定版本
spring-ai-alibaba-dashscope 阿里云百炼平台依赖，调用千问、DeepSeek等模型 ⚠️ 未包含在1.1 BOM中，必须手动指定版本（如3.4.8，需与Nexus版本匹配）
spring-ai-mcp-server-webflux MCP SSE/流式传输服务端依赖，属于Spring AI官方包 Spring AI BOM管理，无需指定版本

二、MCP（模型上下文协议）1.1版本全链路开发

MCP（Model Context Protocol，模型上下文协议）是解决大模型无法获取实时数据、本地资源的通信标准，1.1版本对其做了深度优化，大幅简化开发流程。

2.1 1.1版本MCP核心优化

  • 内置MCP SDK 0.14，无需额外导入底层MCP依赖
  • 传输方式扩展：保留stdio（本地进程通信）、SSE（默认流式传输），新增Streamable HTTP（SSE升级版）、Stateless（无状态流式传输，适配微服务/分布式场景）
  • 全新MCP专用注解，替代通用注解，工具自动注册，无需手动配置ToolCallbackProvider
  • 流式传输统一支持：SSE/Streamable HTTP/Stateless均为流式协议，推荐用Spring WebFlux处理

三、A2A（智能体间通信协议）1.1版本全链路开发

A2A（Agent-to-Agent）协议由Google提出，Spring AI Alibaba 1.1原生支持，实现跨节点、跨模型智能体通信，核心组件如下：

3.1 A2A核心组件

组件 作用 说明
Provider（服务提供者） 提供特定能力的智能体，将自身元数据注册到Nexus 基于ReAct框架，具备自主规划、决策、工具调用、记忆、环境感知能力
Consumer（服务消费者） 需要调用其他智能体能力的智能体，从Nexus发现服务 通过A2aRemoteAgent对象匹配远程智能体
Nexus 3.1注册中心 存储所有智能体的元数据（AgentCard），支持MCP、智能体管理 Docker启动，默认端口8088，账号密码均为nexus

3.2 AgentCard（智能体名片）

AgentCard是智能体的“身份证”，存储在Nexus注册中心，是服务匹配的核心依据：

字段 说明 注意事项
name 智能体唯一标识 必填，服务消费者通过name匹配对应智能体，必须与Provider的智能体名称一致
description 智能体能力描述 消费者根据描述选择服务
url 智能体服务地址 Nexus通过此地址调用远程智能体
version 智能体版本 -
skills 智能体支持的技能/工具列表 -

四、1.1版本常见坑点汇总

  1. dashscope依赖未包含在1.1 BOM中，必须手动指定版本，否则报错
  2. MCP注解扫描必须开启spring.ai.mcp.annotation.enabled=true，否则@McpTool不生效
  3. 非SSE的MCP传输方式（Streamable HTTP/Stateless）必须在配置文件中声明transport类型
  4. Jmanus导入SSE协议的MCP服务，地址必须带/sse后缀，否则导入失败
  5. A2A调用必填：description、instruction、shareState=true，缺少任意一项会报错
  6. AgentCard的name必须与Provider的智能体名称完全一致，否则服务发现失败
  7. Provider和Consumer必须使用不同端口，且IDEA需允许多实例运行，否则端口冲突
  8. 1.1版本MCP已内置，无需额外导入spring-ai-mcp依赖

总结

Spring AI Alibaba 1.1版本围绕智能体开发核心，大幅简化了MCP、A2A协议的开发流程，开发者可基于原生支持的ReAct智能体框架快速实现多智能体协同、跨节点通信的复杂AI应用，建议优先基于1.1版本开发，后续平滑适配2.0版本迭代。
