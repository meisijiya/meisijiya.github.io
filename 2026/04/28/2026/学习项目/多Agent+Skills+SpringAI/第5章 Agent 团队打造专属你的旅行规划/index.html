第五章 旅行规划多智能体系统设计与协作

本章讲解了复合旅行需求的模块化解决方案：将看似简单的旅行规划拆解为多个子需求，为每个子需求匹配专属智能体，最终通过多智能体协作+大语言模型联动，实现兼顾用户偏好、天气因素、成本优化的智能旅行规划。

5-1 旅行规划的复合任务拆解

核心逻辑

旅行规划并非单一需求，而是由四类相互关联的核心需求构成，需先拆解需求再匹配对应功能。

核心内容

  1. 四大核心需求：住宿、交通、景点、费用（四者并非割裂，存在强关联性）

  2. 各需求对应核心动作

    需求类型 核心动作 说明
    景点 ① 景点推荐（结合用户偏好+天气状况） ② 景点路线规划 景点推荐需同时满足用户喜好与实际天气条件（如恶劣天气不推荐户外项目）
    交通 路线规划 包含两类场景：景点之间的交通、景点与住宿之间的交通
    费用 费用计算+折扣监控 费用分为固定、浮动两类： • 固定费用：交通（火车/高铁等无折扣）、景点门票（无折扣） • 浮动费用：住宿+餐饮（可监控折扣，降低总成本）

  3. 设计建议：围绕上述子需求与动作构建功能，实现个性化、可落地的旅行规划。

5-2 团队成员：路线制定专员Agent

定位

负责所有路线规划相关需求的专属智能体，解决跨地点的路线串联问题。

典型场景示例

用户需求：十一国庆从深圳自驾游清远3天，行程不宜过满（用户无明确景点意向，需智能体全链路规划）
该需求可拆解为：景点推荐、住宿选址、景点间路线串联、住宿与景点间路线规划、住宿餐饮折扣监控等子需求。

核心能力

  1. 地点搜索与经纬度坐标转换
  2. 路线规划（支持景点间、住宿与景点间两类场景）
  3. 规划路线地图可视化展示

应用场景

住宿路线规划、景点路线规划

5-3 团队成员：行程规划经理Agent

定位

负责行程整体安排、景点/餐饮推荐的专属智能体，输出可落地的详细行程方案。

核心能力

  1. 网络信息抓取：搜索景点信息、抓取用户评价、下载景点/餐厅配图
  2. 天气查询：结合旅行期间的天气状况调整推荐（如大雨天不推荐爬山、冲浪等项目）
  3. 本地文件操作：读取用户本地存储的偏好信息，实现个性化推荐
  4. 表格制作：输出包含时间、地点、事项、备注的详细行程安排

应用场景

景点推荐（含推荐理由、配图）、餐饮推荐、完整行程制定

5-4 团队成员：费用统筹管家Agent

定位

负责全链路费用统计、折扣监控的专属智能体，提升费用透明度，降低旅行成本。

核心能力

  1. 数学计算：统计住宿、交通、饮食、景点的费用明细与总花费
  2. 表格制作：输出结构化的费用明细表
  3. 网络抓取：监控美团等平台的住宿、餐饮优惠券/折扣信息
  4. 定时脚本制作：持续监控折扣发放，及时抓取优惠

应用场景

交通/景点/住宿费用计算、住宿餐饮折扣监控

补充

最终可由汇总智能体将路线、行程、费用信息整合为包含文字、图片、表格的本地文档，方便用户留存查看。

5-5 Agent团队和大模型的无缝协作

完整智能体团队构成（共5类）

智能体类型 属性 核心职责
规划智能体（主管） 通用（所有需求适用） 识别用户意图，拆解子需求，根据各智能体能力分配对应任务
路线制定专员Agent 专属 负责全场景路线规划
行程规划经理Agent 专属 负责景点/餐饮推荐、行程安排
费用统筹管家Agent 专属 负责费用计算、折扣监控
汇总智能体 通用（所有需求适用） 整合所有智能体的输出结果，生成友好的用户回复或结构化文档

完整工作流

用户提出需求 → 规划智能体（主管）拆解需求 → 分配子任务给对应专属智能体 → 各智能体调用工具执行任务 → 汇总智能体整合结果返回用户

底层协作逻辑

  1. 核心中枢：大语言模型（LLM）
    • 负责用户意图分析、需求拆解、工具调用决策，是整个系统的“大脑”
  2. 智能体核心能力
    • 感知能力：搜索用户周边信息，辅助LLM做出准确分析
    • 计划能力：将LLM返回的任务计划分配给对应专属智能体
    • 记忆能力：记录任务执行进度，避免重复或遗漏步骤
    • 执行能力：调用对应工具完成具体任务
  3. 协作链路
    用户需求 → LLM分析拆解需求 → 主管智能体分配任务 → 各专属智能体向LLM请求工具调用建议 → 调用工具执行任务 → 汇总智能体整合结果

设计建议

按功能拆分需求并配置对应智能体，可大幅提升规划完整性与费用透明度。

5-6 百度地图MCP提供的工具体系全景

  1. 天气查询：根据经纬度查询天气，辅助决定游玩景点。
  2. 地理编码：实现地点名称与经纬度坐标的相互转换。
  3. 地点搜索：搜索指定地点周边的住宿、景点、美食等信息。
  4. 路线规划：提供起点与终点之间驾车、步行、公交等路线规划。
  5. 地图可视化：生成路线规划展示页面，支持二维码或URL分享。

第五章核心总结

本章遵循「需求拆解→单智能体设计→多智能体协作→大模型联动」的递进逻辑，完整落地了旅行规划助手Agent的开发思路：

  1. 先拆解复合旅行需求为景点、交通、费用、住宿餐饮四大类子需求，明确每类的核心动作与成本属性（固定/浮动）
  2. 为每个子需求设计专属智能体，明确各智能体的定位、能力边界与应用场景
  3. 补充2类通用智能体（主管规划、结果汇总）形成完整团队，明确多智能体协作流程
  4. 依托大语言模型作为决策中枢，结合智能体的感知、计划、记忆、执行能力，实现需求的全链路自动化处理
