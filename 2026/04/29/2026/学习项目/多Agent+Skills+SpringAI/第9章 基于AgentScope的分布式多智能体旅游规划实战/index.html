9.1 分布式多智能体架构选型：AgentScope vs 其他框架

课程讲义核心内容

1. 多智能体协同两种设计思路

方案 实现逻辑 适用场景
工作流编排 人工设计协同流程，基于Graph引擎/Spring AI Alibaba 1.1 Flow Agent API 流程固定的复杂任务
Agent自主代理 Agent自主决策协同关系，无需人工设定流程 流程灵活、需自主决策的场景

2. 框架对比：AgentScope vs JManus（Java版Manus）

对比维度 JManus AgentScope
定位 通用Java多智能体框架 企业级、工程化分布式多智能体框架
核心特性 基础自主代理能力 ① Docker跨节点部署 ② 基于Spring AI Alibaba 1.1的ReAct框架Agent（支持任务分解+思考-行动-反思机制） ③ 支持MCP、Skills部署 ④ SubAgent独立异步执行 ⑤ A2A协议服务发现
项目选型结论 ❌ 不选用 ✅ 优先选用，作为旅游规划实战核心框架

官方文档补充

  • Spring AI Alibaba Multi-agent架构：支持工作流编排（Tool Calling、Sequential、Parallel等）和Agent自主协同，通过上下文工程细粒度控制各Agent可见信息。
  • AgentScope定位：面向工程实践的智能体开发框架，核心为ReAct循环（推理+行动），支持分布式、模块化、自主决策。

9.5 主管Agent复杂任务自主分解实现

课程讲义核心内容

任务分解两种方式（均基于ReAct框架）

方式 实现 特点 项目选型
enablePlan 设置enablePlan属性为true 内部调用默认PlanNotebook实例，不可自定义配置 ❌ 不选用
planNotebook 传入自定义PlanNotebook实例到planNotebook()方法 可自定义PlanNotebook属性，灵活配置 ✅ 优先选用

原理说明

enablePlan本质内部调用PlanNotebook.builder().build()默认实例，无自定义参数，因此灵活性不足。

官方文档补充

  • 快速启用计划可直接设置enablePlan()，自定义配置需构建PlanNotebook实例传入。

9.6 任务分解核心：PlanNotebook详解

课程讲义核心内容

1. 定位与能力

PlanNotebook是Agent自主规划与执行的核心对象，包含两大能力：

  • 自主规划（Plan）：复杂任务分解、子任务列表生成
  • 自主决策（ReAct）：思考-行动-反馈循环、动态调整、状态跟踪

2. 核心功能

任务分解、状态跟踪（子任务完成情况）、动态调整（执行不符合目标时修正）、任务完成判断。

3. 可自定义属性

属性 说明 默认值
needUserConfirm 计划步骤是否需要用户确认 false
maxSubtasks 子任务最大数量 无限制
storage 计划存储方式 内存存储

9.7 计划执行可观测性：Hook事件拦截

课程讲义核心内容

1. Hook作用

拦截智能体生成、执行计划的内部事件，破除黑箱，可打印推理过程、工具调用等关键信息。

2. 核心事件类型

事件 说明
PreCallEvent 用户输入事件
PostReasoningEvent Agent推理思考事件
PreActingEvent 准备调用工具事件
PostActingEvent 工具调用完成事件

3. 配置步骤

  1. 实现Hook接口，重写onEvent方法，匹配对应事件并打印
  2. 在PlanNotebook中注册Hook
  3. 公共ReActAgent工具类设置stream=true
  4. 注释智能体执行后的响应打印代码，避免重复输出

9.8 主管Agent任务自主分发实现

课程讲义核心内容

1. 分发逻辑

主管Agent根据子任务性质，匹配对应专长的Agent（如自驾游路线任务分发给路线制定Agent，景点规划任务分发给行程规划Agent），ReAct框架可自主完成匹配，无需人工指定。

2. 能力获取方式演进

阶段 实现方式 缺点
初始阶段 通过Prompt告知主管各Agent的能力描述 需人工维护，扩展性差
优化阶段 通过A2A协议自动发现Nacos中的AgentCard，获取各Agent能力 无需人工配置，自动感知新上线的Agent

官方文档补充

  • AgentScope Supervisor模式：中心监督者接收用户请求，通过工具调用专职Agent（每个Agent注册为一个工具），整合结果返回用户，适合统一入口的任务分派。
  • Agent as Tool：将子Agent注册为工具，固定名称与签名，监督者直接调用。

9.9 基于Nacos+A2A协议的智能体注册与发现

课程讲义核心内容

1. A2A协议角色划分

角色 对应Agent 职责
服务端（服务提供者） 路线制定Agent、行程规划Agent 构建AgentCard，自动注册到Nacos
客户端（服务消费者） 主管Agent 从Nacos发现AgentCard，获取远程Agent能力

2. 服务端实现步骤

  1. 引入A2A依赖、Nacos依赖
  2. 配置application.yml：开启A2A服务、配置Nacos地址
  3. 通过AgentScopeA2AServer构建AgentCard，卡片名称必须与Agent名称一致（Nacos按名称搜索）
  4. 自动注册AgentCard到Nacos

9.10 远程Agent封装为工具执行子任务

课程讲义核心内容

1. 实现目标

主管Agent将发现的远程Agent封装为本地工具，直接调用执行子任务。

2. 实现步骤

  1. 主管模块创建工具包，定义工具方法，加@Tool注解
  2. 通过A2A协议从Nacos获取远程Agent的AgentCard
  3. 工具方法内实现远程Agent调用逻辑（阻塞获取结果）
  4. 将远程调用逻辑抽象到公共模块Nacos工具类，供各模块复用
  5. 将工具注册到主管Agent的工具包，主管可直接调用远程Agent

3. 执行逻辑

主管调用工具 → 远程Agent执行子任务 → 返回结果 → 主管继续后续流程

本章最佳实践汇总

✅ 企业级分布式多智能体优先选用AgentScope，充分利用其工程化特性
✅ 任务分解优先使用planNotebook方法而非enablePlan，支持自定义配置
✅ PlanNotebook建议设置子任务最大数量（如5个），避免任务过于碎片化
✅ Hook拦截建议开启，便于调试与观测执行过程
✅ AgentCard名称必须与Agent名称一致，否则Nacos无法发现
✅ 远程Agent调用逻辑抽象到公共工具类，避免代码重复
✅ 同一Agent实例不可并发调用，多请求场景需创建独立实例
