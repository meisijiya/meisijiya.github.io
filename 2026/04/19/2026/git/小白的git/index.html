个人开发基本git命令

git对于小白来说无非就是起到一个版本管理的作用（ 存档），以至于多人协作等高阶玩法，慢慢摸索即可。

因此，本篇博客站在一个小白的视角，看看如何才能快速上手git。

首先，当然是要从安装git说起，下载地址：https://git-scm.com/；下载下来后安装程序就是无脑选择推荐选项和点击下一步即可（当然可以勾选上On the Desktop选项）。

安装好后，就可以进入到项目中初始化git仓库，通过git init初始化即可，当然也可通过git clone [url]克隆远程仓库到执行命令的目录下。

本地仓库设置好了，我们怎么关联远程仓库呢，首先得去GitHub创建一个仓库，然后我们再通过git remote set-url origin [url]，这个命令会将本地仓库关联到远程仓库

通过IDE打开项目后，可以通过IDE内置的终端进行git命令操作。

当我们完成一个功能后，就会习惯性的提交一下功能实现的代码，使用git add .就是将工作目录下的所有文件添加进暂存区中，可以通过git add [file]来指定文件添加进暂存区。

可以通过git status查看当前工作区和暂存区的状态，示例如下：

当完成一个大模块的时候，会发现提交了好多次记录，想要查看历史提交git log，这时候可能会想着合并模块实现的所有请求，那么可以使用变基（rebase）操作git rebase -i HEAD~n，对当前分支最近 n 个提交，进行交互式整理（合并、修改、删除、重命名），让提交历史干净整洁。

merge是啥，merge就是把 两个不同分支 合并在一起。

然后esc，并输入:wq，Git就会自动开始合并提交。

那如果上一次提交差了点东西，仅仅只是想将暂存区的更改追加到上一次提交中呢，那么先将更改添加到暂存区git add .，然后再git commit --amend -- no-edit将暂存区的更改合并到上一次提交且不修改提交信息。

当然，想要修改提交信息，git commit --amend就会打开一个编辑器让你修改提交信息，保持关闭即可。

那肯定会有小白不小心误提交了，那怎么办，执行git reset --soft HEAD~n，这个命令就是撤销n个新提交但保留修改。

当远程仓库的分支和现在本地分支进度不一致或者修改了已推送的历史，那么可能需要强制推送了，这里面风险很大，但是因为我们都是个人使用不存在多人协作，所有强制推送对我们没啥事，执行git push --force-with-lease进行强制推送。
